Tenth Blog: Metadata

Metadata in the context of computers is a prefix that mean “an underlying description”; metadata is data that is used to describe other data. It is used to give a summary of the content/data in a file, and therefore can be defined as the references to data. It exists to give the context of the webpage, examples of metadata include, author, file size, the date etc.

Metadata is used for images, video and webpages. In a web page, the metadata is usually placed in the head of the html document. They can contain links to external resources/documents used in the web page, so one can link a CSS file and a JavaScript file inside the HTML file.

There are three types of metadata, the first one is the descriptive metadata; this metadata explains what things mean, who the author of the data is, it gives readers the idea of what the web page contains, and when the data was published. There is structural data, this data explains how the data relates to other data sets in the page, it explains how the data is organised and whether the data is sorted or not.

Administrative metadata is data that explains when the data in the document was created, changed and collected, the size of the data and who is granted access to that data

According to Jason Hare (2016), discussions around data quality and data discoverability revolve around metadata and ontologies. Ontologies are descriptions and definition of relationships, examples of ontologies include classes, functions, instances etc; and these help us to understand that a Mac computer is a subject of an object class, cell phone.
